## A

### AI-Assisted Development
**AI 辅助开发**：使用 AI 工具辅助软件开发的方法，包括代码生成、代码补全、代码审查等。Vibe Coding 是 AI 辅助开发的一种特定方法。

### Artifact
**产出物**：AI 生成的具体成果，如代码、文档、图表等。在 Claude 等工具中，Artifact 指可以独立展示和使用的生成内容。

---

## C

### Context Window
**上下文窗口**：AI 模型一次能处理的最大文本量。例如 Claude 的上下文窗口是 200K tokens，意味着它能"记住"约 15 万字的对话内容。上下文窗口越大，AI 能理解的信息越多。

### Co-Pilot
**副驾驶**：指 AI 作为辅助角色，帮助开发者编程。GitHub Copilot 是最著名的代码副驾驶工具。在 Vibe Coding 中，AI 更像是"协作伙伴"而非单纯的"副驾驶"。

---

## F

### Few-Shot Learning
**少样本学习**：通过提供少量示例让 AI 理解任务的方法。例如给 AI 2-3 个示例，它就能理解你想要的代码风格。

### Flow State
**心流状态**：完全沉浸在工作中的状态，时间感消失，效率极高。Vibe Coding 的目标之一是帮助开发者更容易进入心流状态。

---

## I

### Incremental Refinement
**增量优化**：通过小步迭代逐步改进代码，而不是一次性重写。Vibe Coding 的核心实践之一。

### Iteration
**迭代**：重复执行"实现 → 测试 → 反馈 → 改进"的循环。Vibe Coding 强调快速、小步的迭代。

---

## L

### LLM (Large Language Model)
**大语言模型**：通过大量文本训练的 AI 模型，能理解和生成自然语言。Claude、GPT-4 等都是 LLM。Vibe Coding 依赖 LLM 的对话和代码生成能力。

---

## P

### Pair Programming
**结对编程**：两个程序员一起工作，一个写代码（驾驶员），一个审查和指导（导航员）。Vibe Coding 可以看作是"人与 AI 的结对编程"。

### Prompt
**提示词**：给 AI 的输入指令或问题。好的 Prompt 能让 AI 生成更符合预期的输出。

### Prompt Engineering
**提示词工程**：设计和优化 Prompt 的技术。在 Vibe Coding 中，约束引导技术就是一种 Prompt Engineering。

---

## R

### Rapid Prototyping
**快速原型**：快速构建可用原型以验证想法的方法。Vibe Coding 特别适合快速原型开发。

### Refactoring
**重构**：在不改变功能的前提下改进代码结构。Vibe Coding 强调"先实现，后重构"。

---

## T

### Temperature (AI 参数)
**温度参数**：控制 AI 输出随机性的参数。温度越高，输出越有创意但可能不准确；温度越低，输出越确定但可能缺乏创意。通常代码生成使用较低温度。

### Token
**令牌**：AI 处理文本的基本单位。一个 token 大约是 0.75 个英文单词或 0.5 个中文字。Token 数量影响 API 成本和上下文窗口限制。

---

## V

### Vibe Coding
**感觉驱动编程**：一种新的编程范式，强调通过与 AI 对话、快速迭代、基于"感觉"判断来开发软件，而不是传统的详细规划和精确实现。

### Vibe Check
**感觉检查**：在 Vibe Coding 中，通过直觉和整体感受判断代码或方案是否"对"的过程。不是玄学，而是基于经验的快速整体判断。

---

## Z

### Zero-Shot Learning
**零样本学习**：不提供示例，直接让 AI 完成任务。例如直接问"写一个排序函数"，AI 能理解并生成代码。

---

## 中文术语

### 感觉驱动编程
Vibe Coding 的中文译名。强调基于"感觉"（直觉、整体判断）而非详细规划的编程方式。

### 快速反馈循环
**Fast Feedback Loop**：从行动到获得反馈的时间很短的循环。Vibe Coding 的核心是建立快速反馈循环（分钟级而非天级）。

### 许愿式编程
**Wishful Programming**：一种反模式，指把需求告诉 AI 就期待自动生成完美代码，自己什么都不管。这是 Vibe Coding 要避免的陷阱。

### 约束引导
**Constraint Guidance**：通过给 AI 明确的约束条件（技术栈、代码风格、性能要求等）来引导其生成符合预期的代码。Vibe Coding 的核心技巧之一。

### 增量修改
**Incremental Modification**：每次只修改一小部分代码，测试后再继续。与"推倒重来"相对，是 Vibe Coding 推荐的方式。

### 检查点机制
**Checkpoint Mechanism**：在关键节点停下来验证和审查的机制。帮助及时发现问题，避免走弯路。

### 分层控制
**Layered Control**：在 Vibe Coding 中，人和 AI 各司其职：人控制战略层（架构、核心逻辑、关键决策），AI 负责执行层（代码生成、重复工作）。
