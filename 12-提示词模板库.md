## 1. 需求描述模板

### 模板 1.1：功能开发

```
我需要[功能描述]，类似于[举例]。

但我不确定：
- 技术上是怎么实现的？
- 需要用到哪些技术栈？
- 有没有现成的库或工具？
- 大概的实现思路是什么？

先帮我梳理思路，别急着写代码。
```

---

### 模板 1.2：问题解决

```
我遇到了[问题描述]。

当前情况：
- 预期行为：[描述]
- 实际行为：[描述]
- 错误信息：[如果有]
- 已尝试的方法：[描述]

请帮我分析可能的原因和解决方案。
```

---

### 模板 1.3：学习新技术

```
我想学习[技术名称]，目的是[使用场景]。

我的背景：
- 熟悉：[已掌握的技术]
- 不熟悉：[不了解的领域]

请用我能理解的方式解释：
- [技术名称]是什么？
- 为什么要用它？
- 基本概念和术语
- 如何快速上手？
```

---

## 2. 技术咨询模板

### 模板 2.1：方案选择

```
我想做[功能描述]，但我不懂技术细节。

请告诉我：
1. 需要考虑哪些关键问题？
2. 有哪些常见方案？
3. 每个方案的优缺点是什么？
4. 对于[我的具体场景]，你推荐哪种？
```

---

### 模板 2.2：技术解释

```
你刚才提到[技术术语]，我不太理解。

能用简单的语言解释：
- 这是什么？
- 为什么要用它？
- 不用它会怎样？
- 有没有更简单的替代方案？
```

---

### 模板 2.3：架构设计

```
我需要设计[系统/模块]的架构。

需求：
- 功能：[列出主要功能]
- 约束：[性能、规模、技术栈等]
- 未来扩展：[可能的变化]

请提供：
1. 整体架构建议
2. 关键模块划分
3. 技术选型建议
4. 需要注意的坑
```

---

## 3. 代码审查模板

### 模板 3.1：代码质量检查

```
请审查以下代码：

[粘贴代码]

重点检查：
- 是否有明显的 bug？
- 性能问题？
- 安全隐患？
- 代码风格和可读性？
- 有没有更好的实现方式？
```

---

### 模板 3.2：重构建议

```
这段代码能工作，但我觉得不够优雅：

[粘贴代码]

请建议：
- 如何重构能让它更清晰？
- 有没有设计模式可以应用？
- 如何提升可维护性？

注意：保持功能不变。
```

---

## 4. 调试协助模板

### 模板 4.1：错误诊断

```
代码报错了：

错误信息：
[粘贴完整错误信息]

相关代码：
[粘贴代码]

环境信息：
- [语言/框架版本]
- [操作系统]
- [其他相关信息]

请帮我：
1. 解释这个错误是什么意思
2. 可能的原因
3. 如何修复
```

---

### 模板 4.2：行为异常

```
代码没报错，但行为不对：

预期：[描述预期行为]
实际：[描述实际行为]

相关代码：
[粘贴代码]

测试数据：
[如果有]

请帮我找出问题所在。
```

---

## 5. 学习引导模板

### 模板 5.1：概念理解

```
我在学习[主题]，遇到了[概念]不太理解。

我的理解是：[你的理解]

请帮我：
1. 纠正我理解错误的地方
2. 用类比或例子解释这个概念
3. 它在实际中如何应用？
```

---

### 模板 5.2：最佳实践

```
关于[技术/场景]，业界的最佳实践是什么？

具体想了解：
- 常见的做法有哪些？
- 各有什么优缺点？
- 什么情况下用哪种？
- 有哪些常见的坑？
```

---

## 6. 迭代修改模板

### 模板 6.1：增量修改

```
基于当前代码，请做以下修改：

1. [具体修改点 1]
2. [具体修改点 2]
3. [具体修改点 3]

注意：
- 只修改这些地方，其他保持不变
- 保持代码风格一致
```

---

### 模板 6.2：功能增强

```
当前代码实现了[现有功能]。

现在需要增加：
- [新功能 1]
- [新功能 2]

要求：
- 不破坏现有功能
- 保持代码结构清晰
- [其他约束]

请先说明你的实现思路，再写代码。
```

---

## 7. 场景专用模板

### 模板 7.1：快速原型

```
我需要快速验证[想法/功能]。

核心需求：
- [最小可用功能]

不需要：
- 完善的错误处理
- 性能优化
- 完整的测试

请给出最简单能跑的实现。
```

---

### 模板 7.2：生产代码

```
我需要生产级别的[功能]实现。

要求：
- 完善的错误处理
- 性能考虑
- 安全性
- 可维护性
- [其他生产要求]

请给出完整、健壮的实现。
```

---

## 8. 约束引导模板

### 模板 8.1：技术约束

```
请实现[功能]，必须遵循以下约束：

技术栈：
- 语言：[指定语言]
- 框架：[指定框架]
- 库：[允许/禁止的库]

代码风格：
- [命名规范]
- [代码组织方式]
- [其他风格要求]

性能要求：
- [具体指标]
```

---

### 模板 8.2：业务约束

```
实现[功能]时，需要考虑：

业务规则：
- [规则 1]
- [规则 2]

边界情况：
- [情况 1]如何处理
- [情况 2]如何处理

用户体验：
- [UX 要求]
```

## 9. 组合使用示例

### 示例 1：从零开始的完整流程

```
第一步 - 技术咨询：
"我想做[功能]，但不懂技术细节。请告诉我..."

第二步 - 确认方案：
"明白了，那就用[方案]。现在请..."

第三步 - 代码实现：
"按照刚才讨论的方案，请实现..."

第四步 - 审查优化：
"代码能跑了，请审查一下..."
```

---

### 示例 2：问题解决流程

```
第一步 - 问题描述：
"我遇到了[问题]，当前情况是..."

第二步 - 深入诊断：
"你提到可能是[原因]，能详细解释吗？"

第三步 - 实施修复：
"明白了，请帮我修复..."

第四步 - 验证测试：
"修复后，如何验证问题解决了？"
```

## 10. 模板使用技巧

### 技巧 1：根据场景调整

```
不要：直接复制粘贴
而是：根据实际情况调整

调整内容：
- 技术栈
- 功能需求
- 约束条件
- 质量标准

保持模板结构，修改具体内容
```

### 技巧 2：组合使用

```
复杂任务可以组合多个模板：

例子：实现完整功能
1. 用"技术咨询模板"讨论方案
2. 用"功能开发模板"实现功能
3. 用"代码审查模板"检查质量
4. 用"优化模板"改进代码

组合使用，覆盖全流程
```

### 技巧 3：建立个人模板库

```
基于这些模板，建立自己的：

1. 记录常用的提示词
2. 调整为适合自己的风格
3. 添加个人偏好
4. 持续优化

形成个人武器库
```

### 技巧 4：版本管理

```
模板也需要版本管理：

记录：
- 创建时间
- 使用次数
- 效果如何
- 改进历史

定期更新，保持有效
```
