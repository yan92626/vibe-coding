## 1. AI 编程工具的演进历程

### 1.1 我的工具使用时间线

**2023年初-中期：Web 对话时代**
- 工具：GPT-3.5 web 界面
- 痛点：反复复制粘贴，上下文丢失，无法持久化
- 解决：手动维护文档，频繁保存对话
- 深层问题：token 限制（4K）导致无法包含完整项目上下文，每次对话都是"失忆"的

**2023年中-下期：API 调用时代**
- 工具：OpenAI API、DeepSeek、Doubao
- 痛点：不同模型 API 格式不统一，function calling 调试困难
- 解决：封装统一的 API 客户端，标准化 schema 设计
- 深层问题：function calling 的 schema 设计是个艺术，参数太多模型理解不了，太少又不够用；错误处理机制不完善，调试像黑盒

**2024年上半年：Prompt Engineering 时代**
- 技术：Zero-shot、Few-shot、CoT、ReAct
- 痛点：prompt 不稳定，缺乏系统化方法论
- 解决：建立 prompt 模板库，A/B 测试不同版本
- 深层认知：这个阶段意识到"prompt 工程"本质是"沟通工程"——不是研究技巧，而是学会清晰表达意图；同样的需求，不同的表达方式，效果天差地别

**2024年中期：可视化编排时代**
- 工具：Coze、Dify
- 痛点：灵活性受限，复杂逻辑难实现，难以版本控制
- 解决：转向代码化的 agent 框架
- 深层反思：可视化编排的本质问题是"抽象层次错位"——它试图用低代码方式解决高复杂度问题，结果是简单场景过度复杂，复杂场景力不从心；真正需要的是"代码 + 配置"的混合模式

**2024年下半年：Agent 框架时代**
- 工具：LangChain、LangGraph、CrewAI
- 痛点：学习曲线陡峭，框架更新快，agent 协调复杂
- 解决：简化架构，只用必要的功能
- 深层教训：框架的抽象是双刃剑——它降低了入门门槛，但也限制了灵活性；更重要的是，很多"agent 协调"的复杂性是人为制造的，一个好的 prompt 往往比三个 agent 更有效

**2024年末-2025年初：IDE 集成时代**
- 工具：Cursor、Copilot、Claude Code
- 痛点：工具太多不知道选哪个，如何让 AI 理解项目上下文
- 解决：建立工具组合策略，用 CLAUDE.md 等配置文件
- 深层理解：这个阶段的关键突破是"上下文管理"——不是简单地"把代码给 AI"，而是智能地管理什么时候给什么；Cursor 的 @file、Claude Code 的自动上下文压缩，本质都在解决"如何高效利用有限的上下文窗口"

**2025年-2026年初：MCP/Skill 时代**
- 技术：MCP、Skill、Harness
- 思考：如何设计可复用的能力，如何在工具间迁移经验
- 方向：建立跨工具的方法论，而不是依赖特定工具
- 深层洞察：这个阶段意识到"能力封装"的重要性——Skill 不只是 prompt 模板，而是"意图 + 上下文 + 约束"的完整封装；MCP 不只是协议，而是"能力标准化"的尝试；Harness 概念揭示了"AI 工具的本质是人机协作的脚手架"

**关键转折点的反思**：

从 API 到 Prompt Engineering：意识到"如何说"比"说什么"更重要

从可视化到代码框架：理解了"抽象层次"的重要性，不是所有东西都适合可视化

从框架到 IDE 集成：发现"工作流集成"比"功能强大"更重要，打断越少效率越高

从工具到方法论：最终认识到工具会变，但思维方式不变——这就是 Vibe Coding 的价值

### 1.2 工具选择的演进思考

这个演进历程让我意识到：

1. **工具会变，方法论不变**：从 GPT-3.5 到 Claude 4，工具在进化，但"清晰表达意图"的原则不变
2. **不要被工具绑架**：每个新工具都声称"革命性"，但核心还是人机协作
3. **简单优于复杂**：从复杂的 agent 框架回归到简单的对话，往往更有效
4. **适合自己最重要**：不是最新的工具就最好，而是最适合你的工作流的工具最好

这正是本文档的价值所在：不是教你用某个具体工具，而是教你一套可以跨越工具变迁的思维方式。

## 2. 主流 AI 编程工具对比

### 2.1 Claude (Code/Web)

**特点**：
- 对话能力强，理解复杂需求
- 代码质量高
- 擅长解释和教学
- 上下文理解深入

**优势**：
复杂技术讨论
学习新技术
架构设计
代码审查
深入解释
可接入第三方模型API

**劣势**：
免费版有使用限制
响应速度相对较慢
价格很贵

**适用场景**：
- 学习新技术栈
- 讨论技术方案
- 代码审查和优化
- 需要详细解释的场景
- 复杂业务逻辑设计

**价格**：
- 免费版：有使用限制
- Pro 版：$20/月
- Claude Code：包含在 Pro 中

**推荐指数**：⭐⭐⭐⭐⭐

---

### 2.2 ChatGPT

**特点**：
- 响应速度快
- 知识面广
- 插件生态丰富
- 多模态能力

**优势**：
快速查询
简单问题
多语言支持
插件扩展
图片理解
可接入第三方模型API

**劣势**：
代码质量不如 Claude

**适用场景**：
- 快速查询错误
- 简单代码生成
- 文档编写
- 多语言翻译
- 需要插件功能

**价格**：
- 免费版：GPT-3.5
- Plus：$20/月（GPT-4）
- Team：$25/月/人
- Enterprise：定制

**推荐指数**：⭐⭐⭐⭐

---

### 2.3 GitHub Copilot

**特点**：
- IDE 无缝集成
- 实时代码补全
- 上下文理解好
- 不打断工作流

**优势**：
代码补全快速
IDE 集成完美
上下文感知
支持多种语言
不打断流程

**劣势**：
对话能力弱
不适合讨论方案
需要付费
依赖网络

**适用场景**：
- 日常编码
- 代码补全

- 标准模式代码
- IDE 内开发

**价格**：
- 个人：$10/月
- 企业：$19/月/人
- 学生/开源：免费

**推荐指数**：⭐⭐⭐⭐

---

### 2.4 Cursor

**特点**：
- 专为 AI 编程设计的 IDE
- 集成 AI 对话和编辑
- 项目上下文理解
- 多模型支持

**优势**：
AI 原生 IDE
对话 + 编辑一体
项目级理解
多模型切换
快捷键丰富

**劣势**：
需要付费

**适用场景**：

**推荐指数**：⭐⭐⭐⭐⭐

## 3. 选择标准

### 3.1 根据任务类型选择

**学习和探索**：
- 首选：Claude
- 备选：ChatGPT
- 原因：解释清晰，适合学习

**日常开发**：
- 首选：Cursor 或 Copilot
- 备选：Claude Code
- 原因：IDE 集成，效率高

**快速查询**：
- 首选：ChatGPT
- 备选：Claude
- 原因：响应快

**代码审查**：
- 首选：Claude
- 备选：ChatGPT
- 原因：分析深入

---

### 3.2 根据技术栈选择

**前端开发**：
- Cursor + Copilot
- 原因：组件补全好

**后端开发**：
- Claude + Copilot
- 原因：逻辑复杂，需要讨论

**数据分析**：
- Claude + ChatGPT
- 原因：需要解释和可视化

**DevOps**：
- ChatGPT + Copilot
- 原因：脚本生成快

---

### 3.3 根据工作流选择

**探索阶段**：
- Claude（讨论方案）
- 原因：需要深入讨论

**实现阶段**：
- Cursor/Copilot（编码）
- 原因：快速实现

**调试阶段**：
- ChatGPT（快速查询）
- Claude（深入分析）

**优化阶段**：
- Claude（代码审查）
- 原因：提供改进建议

---

### 3.4 根据预算选择

**预算充足（$40+/月）**：
- Cursor Pro + Claude Pro
- 最佳组合，覆盖所有场景

**中等预算（$20-40/月）**：
- Claude Pro 或 Cursor Pro
- 选一个主力工具

**低预算（$10-20/月）**：
- Copilot + Claude 免费版
- 基本够用

**零预算**：
- Claude 免费版 + Codeium
- 功能受限但可用

## 5. 工具使用技巧

### 5.1 Claude 使用技巧

**技巧 1：结构化提问**
```
不要："帮我做个网站"
而是："我想做 [具体描述]
       技术栈：[明确]
       要求：[列出]"
```

**技巧 2：分步讨论**
```
第 1 步：讨论方案（不写代码）
第 2 步：确定方案
第 3 步：实现代码
第 4 步：审查优化
```

**技巧 3：利用长上下文**
```
- 提供完整代码
- 详细描述问题
- 充分讨论方案
```

---

### 5.2 ChatGPT 使用技巧

**技巧 1：快速查询**
```
"这个错误是什么意思？
[粘贴错误信息]"

快速获得答案
```

**技巧 2：使用插件**
```
- 代码解释器
- 网页浏览
- 图片生成
```

**技巧 3：多轮对话**
```
逐步细化需求
快速迭代
```

---

### 5.3 Copilot 使用技巧

**技巧 1：写好注释**
```
// 创建一个函数，验证邮箱格式
// 参数：email 字符串
// 返回：布尔值

Copilot 会根据注释生成代码
```

**技巧 2：利用上下文**
```
Copilot 会参考：
- 当前文件
- 相关文件
- 项目结构

保持代码整洁，提升补全质量
```

**技巧 3：快捷键**
```
Tab：接受建议
Alt+]：下一个建议
Alt+[：上一个建议
Ctrl+Enter：打开建议面板
```

---

### 5.4 Cursor 使用技巧

**技巧 1：Cmd+K 编辑**
```
选中代码
Cmd+K
输入修改要求
快速修改
```

**技巧 2：Cmd+L 对话**
```
Cmd+L 打开对话
讨论方案
生成代码
```

**技巧 3：@符号引用**
```
@file：引用文件
@code：引用代码
@docs：引用文档

提供精确上下文
```
